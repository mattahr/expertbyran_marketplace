# Intervjumetodik — referensmaterial

_Semi-strukturerade intervjuer i effektivitetsrevision: urval, guidekonstruktion, kodning, konvergens, COREQ och analytisk generalisering._

---

## Urvalsstrategi

**Norm:** Strategiskt urval (purposive sampling) — maximera variation på teoristyrda dimensioner, inte statistisk representativitet.

**Princip:** Vid granskning av myndighetsutövning: välj informanter med direkt erfarenhet av beslutssituationen, inte bara övergripande chefer. Chefsintervjuer ger policylogik; handläggarintervjuer ger implementeringsverkligheten.

**Urvalsprotokoll:**
1. Identifiera relevanta aktörsnivåer utifrån granskningsfrågan (t.ex. policy → implementering → mottagare).
2. Bestäm variationsdimensioner (t.ex. kommunstorlek, geografisk region, typ av verksamhet).
3. Välj fall som maximerar variation längs dessa dimensioner.
4. Dokumentera bortfall och motivera hur det hanteras.

---

## Intervjuguide — tre nivåer

**Nivå 1: Öppna huvudfrågor** (max 5–7 per guide)
- Breda, icke-ledande. Låter informanten rama in sin upplevelse.
- Exempel: "Kan du berätta om hur ni hanterar ansökan om statsbidrag?"

**Nivå 2: Uppföljningssonder**
- Fördjupar utan att styra: "Kan du ge ett konkret exempel?", "Vad menar du med...?", "Hur brukar det gå till?"
- Sondar är inte förbestämda — de beror på vad informanten berättar.

**Nivå 3: Bakficka med faktafrågor**
- Specifika faktafrågor för triangulering mot dokumentkällor.
- Ställs mot slutet, om inte informanten redan tagit upp det.
- Exempel: "Årsredovisningen 2023 anger att ni följer upp bidraget kvartalsvis — stämmer det med din bild?"

**Guideprincipen:** Guiden är ett stöd, inte ett manus. Avvik om informanten tar upp något metodologiskt viktigt.

## Dokumentation och reflexivitet

**Om möjligt:** Spela in och transkribera.

**Om inspelning inte är möjlig:** Anteckna direkta citat inom 30 minuter efter intervjun. Ange vilka delar som är direkta citat och vilka som är sammanfattningar.

**Reflexivitetsnotering per intervju:** Skriv 3–5 meningar om vad som färgade samtalet — informantens tonläge, eventuell bias, ämnen som undveks, relationens karaktär. Dessa noteringar används vid kodning och metodredovisning.

## Kodning — två pass

### Pass 1: Öppen kodning
- Nära texten, inga förutfattade kategorier.
- Koda allt som kan vara relevant för granskningsfrågan.
- Generera ett initialt kodträd.

### Pass 2: Axiell kodning
- Gruppera koder till teman och identifiera mönster.
- Testa kodträdet mot den teoretiska ramen.
- Identifiera avvikare — de är metodologiskt intressanta, inte bara brus.

### Induktiv vs. teoridriven kodning
- **Induktiv:** Koderna växer ur materialet. Passar när granskningsfrågan är explorativ.
- **Teoridriven (deduktiv):** Kodträdet bygger på programteori eller befintliga begrepp. Passar när en specifik logikmodell testas.
- **Abduktiv (vanligast):** Startar induktivt men justeras mot teorin under pass 2.

## Konvergenströskel vid litet n (< 9 intervjuer)

### Problemet
I effektivitetsgranskningar är n = 5–9 intervjuer per tjänstemannanivå inte ovanligt. Det finns ingen universell numerisk mättnadsregel.

### Arbetsmodell för konvergensbedömning

**Steg 1:** Koda intervju 1–3 intensivt → generera initialt kodträd.

**Steg 2:** Koda intervju 4–6 mot kodträdet → räkna andelen kodningar som kräver nya koder.

**Tröskel:** Om < 20% av kodningarna i en ny intervju kräver nya koder = arbetsoperationell konvergens uppnådd.

**Steg 3:** Koda återstående intervjuer och notera avvikare.

**Viktigt:** Vid homogena urval (t.ex. ekonomichefer i kommuner av liknande storlek) kan konvergens nås vid n = 6–8. Vid heterogena urval krävs fler intervjuer.

### Inter-rater vid litet n

Med n < 9: dubbel-koda **alla** intervjuer (inte bara ett subset). Det är hanterbart och metodologiskt motiverat.

- Beräkna procentuell enighet per tema-kategori (inte Cohen's κ, som kan bli instabilt vid få fall).
- Mål: > 75% enighet på tvärparagrafnivå.
- Dokumentera avvikande bedömningar explicit — de är ofta metodologiskt intressanta.
- Underskrid inte målet utan att redovisa det öppet i metodavsnittet.

## COREQ-granskningsmall (Tong, Sainsbury & Craig 2007)

COREQ (Consolidated Criteria for Reporting Qualitative Research) är en 32-punkters checklista i tre domäner. Används som intern kvalitetsgranskning av metodavsnittet.

### Domän 1: Forskarteam och reflexivitet (8 punkter)

| # | Krav |
|---|------|
| 1 | Intervjuarens namn och titel |
| 2 | Yrke vid tidpunkten |
| 3 | Genus |
| 4 | Erfarenhet och träning |
| 5 | Antaganden och teoretiska utgångspunkter |
| 6 | Relation till informanterna |
| 7 | Informanternas kännedom om intervjuarens bakgrund |
| 8 | Intervjuarens kännedom om informanternas bakgrund |

### Domän 2: Studiedesign (15 punkter)

| # | Krav |
|---|------|
| 9 | Metodologisk orientering (grounded theory, fenomenologi, etc.) |
| 10 | Sampling — vem valdes ut och varför? |
| 11 | Metod för att ta kontakt |
| 12 | Urvalsstorlek |
| 13 | Bortfall — hur många kontaktades och hur många deltog? |
| 14 | Intervjumiljö |
| 15 | Närvaro av andra |
| 16 | Intervjutyp (ansikte mot ansikte, telefon, fokusgrupp) |
| 17 | Inspelning (audio/video) |
| 18 | Fältanteckningar |
| 19 | Intervjulängd |
| 20 | Mättnad — hur bedömdes den? |
| 21 | Återkoppling till informanterna |
| 22 | Transkriptionens natur |
| 23 | Datarevision |

### Domän 3: Analys och fynd (9 punkter)

| # | Krav |
|---|------|
| 24 | Dataanalysens natur — hur kodades materialet? |
| 25 | Teman — hur identifierades de? |
| 26 | Programvara |
| 27 | Deltagarkontroll (member checking) |
| 28 | Citat — presenteras de? |
| 29 | Datakongruens (passar data till fynd?) |
| 30 | Klarhet i huvudteman |
| 31 | Klarhet i underteman |
| 32 | Klarhet i avvikande fall |

**Tillämpning:** Gå igenom listan mot det aktuella metodavsnittet. Flagga saknade element. Notera vilka som inte är tillämpliga och motivera varför.

## Analytisk generalisering

_Yin (2018), Firestone (1993). Kvalitativa studier generaliserar inte statistiskt — de generaliserar analytiskt._

### Yins tvåstegsprocess

**Steg 1: Fall → teori.** Studien visar att en teoretisk proposition stämmer (eller inte) inom fallet.

**Steg 2: Teori → ny kontext.** Teorin — inte fallet i sig — är det som kan överföras till andra kontexter med liknande mekanismer.

Generaliseringen är alltså: "Om den teoretiska mekanismen X stämmer i fall A, och kontexten B delar de förutsättningar som aktiverar X, bör X också verka i B."

### Firestones tre typer

1. **Argumentationsbaserad generalisering:** Forskaren presenterar ett explicit argument för överförbarhet.
2. **Tolkningsbaserad generalisering:** Läsaren bedömer om caset överlappar med sin kontext.
3. **Teoribunden generalisering:** Generaliseringen sker via en explicit teoretisk mekanism.

**Tillämpning i Riksrevisionen:** Alltid typ 3 i metodavsnittet — formulera explicit vilken mekanism som generaliseras och vilka kontextvillkor som krävs.

### Formulering av räckviddsangivelse

> "Iakttagelserna gäller de åtta studerade kommunerna. Analytisk generalisering till liknande kommuner (30 000–80 000 invånare, liknande socioekonomisk profil) är möjlig under förutsättning att [mekanism X] är aktiv i dessa kontexter. Statistisk representativitet kan inte hävdas."
