# Riksrevisionens effektivitetsrevisionsprocess — 3 faser, 23 steg

Riksrevisionens interna process är strukturerad i tre faser med totalt 23 steg och ett antal formella besluts- och kvalitetspunkter (seminarium, intern opponent, extern opponent).

## Fas 1 — Förstudie (idé till beslut om granskning)

**Syfte:** Bedöma om en granskningsidé är värd att genomföra; ta fram tillräckligt underlag för ett genomförandebeslut.

| Steg | Innehåll |
|------|----------|
| 1.1 | Granskningsidé formuleras — internt eller från omvärld |
| 1.2 | Problemanalys: är problemet reellt? Vad vet vi redan? |
| 1.3 | Granskbarhetsbedömning: tillgängliga data, sekretesshinder, revisionsgräns |
| 1.4 | Grov avgränsning: vad ska ingå/uteslutas? |
| 1.5 | Intern diskussion och prioritering |
| 1.6 | Beslut om förstudie (om ej direkt genomförandebeslut) |
| 1.7 | **Förstudieseminarium** — internt QA-tillfälle |
| 1.8 | Beslut om granskning (go/no-go) |

## Fas 2 — Granskning (design, insamling, analys)

**Syfte:** Genomföra granskningen i enlighet med ISSAI 300/3000; producera välgrundade iakttagelser och slutsatser.

| Steg | Innehåll |
|------|----------|
| 2.1 | Precisera revisionsfrågan (från idé till specifik, testbar fråga) |
| 2.2 | Fastslå kriterier (normkällan för bedömning) |
| 2.3 | Planera bevisinhämtning (metoder, datakällor, tidplan) |
| 2.4 | Formulera delfrågor med bedömningsgrunder och indikatorer |
| 2.5 | Intern granskning av granskningsplan |
| 2.6 | **Uppläggsseminarium** — internt QA med inbjudna granskare |
| 2.7 | **Designmatris färdigställd** — det formella instrumentet som specificerar delfrågor, bedömningsgrunder, indikatorer och metodval |
| 2.8 | Datainsamling och analys |
| 2.9 | Löpande analytisk dokumentation |
| 2.10 | Iakttagelser formuleras med stöd i bevisning |
| 2.11 | Slutsatser dras mot normkrav |
| 2.12 | Inledande faktaklarering med granskad aktör |
| 2.13 | **Intern opponent** granskar utkast (metodologisk QA) |
| 2.14 | Revidering baserat på opponent |

## Fas 3 — Rapportering (utkast till publicering)

**Syfte:** Producera en rapport som uppfyller Riksrevisionens vetenskapliga och formella krav; inhämta formell faktaklarering; publicera.

| Steg | Innehåll |
|------|----------|
| 3.1 | Fullständigt rapportutkast |
| 3.2 | **Extern opponent** — oberoende metodologisk QA av fristående granskare |
| 3.3 | Revidering efter extern opponent |
| 3.4 | **Rapportseminarium** — internt QA |
| 3.5 | Formell faktaklarering: granskad aktör får kommentera sakpåståenden |
| 3.6 | Revidering efter faktaklarering |
| 3.7 | Riksrevisorns beslut om rapporten |
| 3.8 | Publicering (remitteras till riksdag och granskad myndighet) |
| 3.9 | **Skrivelse från granskad myndighet** — formellt svar till riksdagen |

## Designmatrisen (steg 2.7) — det viktigaste instrumentet

Designmatrisen är det formella dokument som specificerar:

| Kolumn | Innehåll |
|--------|----------|
| Övergripande revisionsfråga | Den fråga granskningen ska besvara |
| Delfrågor | MECE-indelning av revisionsfrågan |
| Bedömningsgrunder | Normkravet som utfallet mäts mot |
| Indikatorer | Vad som mäter om bedömningsgrunden är uppfylld |
| Metodval | Hur indikatorn mäts (dokumentstudie, intervju, register, etc.) |
| Bedömning | Revisionsteamets bedömning av utfall mot bedömningsgrund |

**MECE-kravet på delfrågor:** delfrågor ska vara Mutually Exclusive (icke-överlappande) och Collectively Exhaustive (täcker hela revisionsfrågan utan luckor). Om delfrågor överlappar uppstår risk för dubbelräkning; om de inte täcker helheten uppstår risk för blinda fläckar.

## Seminarierna — QA-logiken

Riksrevisionens process har tre interna QA-tillfällen och ett externt:

1. **Förstudieseminarium (1.7):** bedömer om granskningsidén är värd att genomföra.
2. **Uppläggsseminarium (2.6):** bedömer om designmatrisen är metodologiskt hållbar.
3. **Intern opponent (2.13):** metodologisk granskning av analysutkast.
4. **Extern opponent (3.2):** oberoende granskning — fristående extern granskare utanför organisationen.
5. **Rapportseminarium (3.4):** QA av fullständigt utkast inför beslut.

Den externa opponenten (steg 3.2) är Riksrevisionens starka QA-element i internationell jämförelse. ECA saknar extern opponent; NAO har extern QA via kommersiella utvärderare (hybridmodell).

## Beslutspunkter

- **Beslut om granskning (1.8):** riksrevisorn (eller delegerat) beslutar om granskningsuppdrag.
- **Riksrevisorns beslut om rapporten (3.7):** slutgiltig godkännande innan publicering.
- Under granskningsprocessen kan arbetet avbrytas vid väsentliga metodproblem eller förändringar i granskningsobjektet.

## Aktörerna i processen

- **Projektledare/granskningsledare:** ansvarar för helheten i en enskild granskning.
- **Granskare/utredare:** genomför datainsamling och analys.
- **Metodansvarig:** stödjer designmatrisen och metodologiska val.
- **Intern opponent:** genomför steg 2.13 — intern metodologisk granskning.
- **Extern opponent:** fristående extern granskare, steg 3.2.
- **Riksrevisorn:** tar det formella beslutet om rapporten.

## Effektivitetsrevisorns roll vid Expertbyrån

Expertbyrån ansluter vanligtvis i fas 2 (steg 2.1–2.7: granskningsdesign) eller fas 3 (steg 3.2: extern opponent). Vanligaste uppdrag är designmatris, metodkapitel, extern opponent och iakttagelse–slutsatskedja. Projektledning av hela granskning förekommer.
